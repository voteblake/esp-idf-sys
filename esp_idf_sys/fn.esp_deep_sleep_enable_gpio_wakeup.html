@brief Enable wakeup using specific gpio pins

This function enables an IO pin to wake the chip from deep sleep

@note This function does not modify pin configuration. The pins are configured in esp_sleep_start, immediately before entering sleep mode.

@note You don’t need to care to pull-up or pull-down before using this function, because this will be done in esp_sleep_start based on param mask you give. BTW, when you use low level to wake up the chip, we strongly recommand you to add external registors(pull-up).

@param gpio_pin_mask Bit mask of GPIO numbers which will cause wakeup. Only GPIOs which are have RTC functionality can be used in this bit map. @param mode Select logic function used to determine wakeup condition: - ESP_GPIO_WAKEUP_GPIO_LOW: wake up when the gpio turn to low. - ESP_GPIO_WAKEUP_GPIO_HIGH: wake up when the gpio turn to high. @return - ESP_OK on success - ESP_ERR_INVALID_ARG if gpio num is more than 5 or mode is invalid,
